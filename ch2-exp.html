2.1 مراحل المشروع - Project Phases

لأن المشاريع تعمل كجزء من نظام وتتضمن عدم يقين، فمن الممارسات الجيدة تقسيم المشاريع إلى عدة مراحل.

Because projects operate as part of a system and involve uncertainty, it is good practice to divide projects into several phases.

1. مرحلة المفهوم
Concept Phase

إنشاء نظرة عامة على العمل المطلوب

2. مرحلة التطوير
Development Phase

خطط إدارة المشروع وتقدير أكثر دقة للتكلفة

3. مرحلة التنفيذ
Implementation Phase

تسليم العمل المطلوب وتقديم تقارير الأداء

4. مرحلة الإغلاق
Close-out Phase

توثيق التجارب وقبول المشروع كاملاً

التسليمات - Deliverables

التسليم هو منتج أو خدمة، مثل تقرير فني، جلسة تدريبية، قطعة أجهزة، أو جزء من كود البرمجيات

A deliverable is a product or service, such as a technical report, a training session, a piece of hardware, or a segment of software code

2.2 نماذج دورة حياة المشروع - Project Life Cycle Models

بعض النماذج الشائعة لدورة حياة تطوير الأنظمة تشمل:

Some popular models of an SDLC include:

نموذج الشلال - Waterfall Model

نموذج دورة حياة الشلال له مراحل خطية محددة جيداً من تحليل النظم والتصميم والبناء والاختبار والدعم

Has well-defined, linear stages of systems analysis, design, construction, testing, and support. Assumes requirements will remain stable.

النموذج الحلزوني - Spiral Model

تم تطوير نموذج دورة حياة الحلزون بناءً على تحسينات نموذج الشلال. يعترف بأن معظم البرمجيات تُطور باستخدام نهج تكراري

Developed based on refinements of the waterfall model. Recognizes iterative development approach rather than linear.

النموذج التزايدي - Incremental Model

يوفر نموذج دورة الحياة التزايدية للتطوير التدريجي للبرمجيات التشغيلية، مع كل إصدار يوفر قدرات إضافية

Provides for progressive development of operational software, with each release providing added capabilities.

نموذج النماذج الأولية - Prototyping Model

يستخدم لتطوير نماذج أولية للبرمجيات لتوضيح متطلبات المستخدم للبرمجيات التشغيلية

Used for developing software prototypes to clarify user requirements. Requires heavy user involvement.

نموذج RAD - RAD Model

نموذج دورة حياة RAD يستخدم نهجاً يعمل فيه المطورون مع نموذج أولي متطور

Uses an approach in which developers work with an evolving prototype. Helps produce systems quickly without sacrificing quality.

التطوير الرشيق - Agile Development

الرشيق يعني استخدام طريقة تعتمد على تسلسلات عمل تكرارية وتزايدية تُعرف عادة باسم السبرينت

Uses iterative and incremental work sequences (sprints). Requirements and solutions evolve through collaboration.

Scrum Framework

Product Backlog
Sprint Planning
Sprint (1-4 weeks)
Daily Scrums
Sprint Review
Sprint Retrospective

سكرم هو الطريقة الرائدة للتطوير الرشيق لإكمال المشاريع ذات النطاق المعقد والمبتكر

Scrum is the leading agile development method for completing projects with a complex, innovative scope of work

2.3 مجموعات عمليات إدارة المشاريع - Project Management Process Groups

العملية هي سلسلة من الإجراءات الموجهة نحو نتيجة معينة. يمكن النظر إلى إدارة المشاريع كعدد من العمليات المترابطة.

A process is a series of actions directed toward a particular result. Project management can be viewed as a number of interlinked processes.

البدء - Initiating

تعريف وترخيص مشروع أو مرحلة مشروع

Defining and authorizing a project or project phase

التخطيط - Planning

وضع والحفاظ على مخطط قابل للتطبيق

Devising and maintaining a workable scheme

التنفيذ - Executing

تنسيق الأشخاص والموارد الأخرى

Coordinating people and resources

المراقبة والتحكم - Monitoring & Controlling

مراقبة وقياس التقدم مقابل الخطط

Monitor and measure progress against plans

الإغلاق - Closing

إضفاء الطابع الرسمي على قبول المشروع

Formalizing acceptance of the project

تطبيق مجموعات العمليات - Process Groups Application

1

مرحلة البدء - Initiating Processes

تشمل عمليات البدء تعريف وترخيص مشروع أو مرحلة مشروع

Initiating processes include defining and authorizing a project or project phase

  • تطوير ميثاق المشروع - Develop Project Charter
  • تحديد أصحاب المصلحة - Identify Stakeholders
2

مرحلة التخطيط - Planning Processes

تشمل عمليات التخطيط وضع والحفاظ على مخطط قابل للتطبيق لضمان أن المشروع يلبي احتياجات المنظمة

Planning processes include devising and maintaining a workable scheme to ensure the project addresses the organization's needs

  • خطة إدارة النطاق - Scope Management Plan
  • خطة إدارة الجدولة - Schedule Management Plan
  • خطة إدارة التكلفة - Cost Management Plan
3

مرحلة التنفيذ - Executing Processes

تشمل عمليات التنفيذ تنسيق الأشخاص والموارد الأخرى لتنفيذ الخطط المختلفة

Executing processes include coordinating people and other resources to carry out various plans

  • الحصول على فريق المشروع وتطويره - Acquire and Develop Project Team
  • تنفيذ ضمان الجودة - Perform Quality Assurance
  • توزيع المعلومات - Distribute Information
  • إدارة أصحاب المصلحة - Manage Stakeholders
4

مرحلة المراقبة والتحكم - Monitoring and Controlling

يراقب مدير المشروع والموظفون التقدم مقابل الخطط ويتخذون إجراءات تصحيحية عند الضرورة

Project manager and staff monitor and measure progress against plans and take corrective action when necessary

  • مراقبة أداء المشروع - Monitor Project Performance
  • التحكم في التغييرات - Control Changes
  • ضمان تحقيق أهداف المشروع - Ensure Project Objectives are Met
5

مرحلة الإغلاق - Closing Processes

تشمل عمليات الإغلاق إضفاء الطابع الرسمي على قبول المشروع أو مرحلة المشروع وإنهائه بكفاءة

Closing processes include formalizing acceptance of the project or project phase and ending it efficiently

  • الأنشطة الإدارية - Administrative Activities
  • أرشفة ملفات المشروع - Archive Project Files
  • إغلاق العقود - Close Out Contracts
